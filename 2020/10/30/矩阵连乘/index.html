矩阵连乘
发表于 2020-10-30| 更新于 2020-10-30|算法--动态规划
字数总计:1.7k|阅读时长: 6 分钟

问题描述

给定n个矩阵{A1,A2,…,An}，及m个矩阵连乘的表达式，判断每个矩阵连乘表达式是否满足矩阵乘法法则，如果满足，则计算矩阵的最小连乘次数，如果不满足输出“MengMengDa“。

输入

输入数据由多组数据组成（不超过10组样例）。每组数据格式如下：
第一行是2个整数n (1 ≤ n ≤ 26)和m(1 ≤ m ≤3)，表示矩阵的个数。
接下来n行，每行有一个大写字母，表示矩阵的名字，后面有两个整数r和c，分别表示该矩阵的行数和列数，其中1 < r, c < 100。
第n+1行到第n+m行，每行是一个矩阵连乘的表达式(2<=矩阵个数<=100)。

输出

对于每个矩阵连乘表达式，如果运算不满足矩阵乘法法则的情况（即左矩阵列数与右矩阵的行数不同），则输出“MengMengDa”，否则输出最小矩阵连乘次数。

数据保证结果不超过1e9。

样例

分析

这一题相对于普通的矩阵连乘题目，加了判断输入的矩阵连乘情况是否成立。因为获取的矩阵连乘信息是一个字符串，我们需要将其转为一个个字符，才能获取其具体的行列信息。判断是否可以连乘的过程中，我们需要通过分离出来的每一个字符，也就是矩阵，在之前存储的矩阵数据数组中寻找其行列信息，通过对相邻矩阵列值和行值是否相等来判断是否可乘。而后就可以通过动态规划的方法，从长度为2开始加括号，将每一次计算的连乘数据中最小的存储到定义的数组中，最后可得出最小连乘次数数据。

理论实现

对于样例中的数据，可以发现ACB这个连乘是可行的，对其进行理论实现。
首先建立一个p数组，存储行列值：

然后建立一个sum数组，存储每一个加括号的结果，计算每种加括号的sum情况：

  1. sum[1][1] = 0，sum[2][2] = 0，sum[3][3] = 0
  2. sum[1][2] = sum[1][1] + sum[2][2] + p[0]p[1]p[2] = 5000
    sum[2][3] = sum[2][2] + sum[3][3] + p[1]p[2]p[3] = 25000
  3. sum[1][3] = min(sum[1][1] + sum[2][3] + p[0]p[1]p[3], sum[1][2] + sum[3][3] + p[0]p[2]p[3]) = 7500

sum表格：

sum A C B
A 0 5000 7500
C 0 25000
B 0

代码

代码中易错点

  1. 题目要求多组输入，因此要用while循环，实现多组数据输入
  2. 注意各种数组的长度问题
  3. 判断是否可乘过程中的逻辑问题，isOk这个标志只能一开始为true，碰到不可乘就改为false，反过来的话会造成覆盖的情况

拓展

题目中没有要求输出最后加括号的数组情况，为了实现输出矩阵加括号的结果，需要在记录sum数据时，加一个记录加括号位置的数组，之后通过这个数组里的数据就可以得到加括号的情况。
