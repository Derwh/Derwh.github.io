    Object和JSON，Map的区别
    与JSON

    JSON是一种轻量级文本数据交换格式，用js编写，但独立于这种语言。

    • JSON => js对象

    • js对象 => JSON

    与Map

    两者都是保存键值对，但是也有区别；

    • Object的键通常是字符串，而Map可以是任意值
    • Map中的键值是有序的
    • Map中键值的个数可以直接获取，Object则要借助Object.keys()等计算得到
    • Map可以之间进行迭代，Object则要借助Object.keys()
    • Map不存在键名和原型键名冲突的情况，可以直接覆盖。

    从某种程度上说，Map比Object更灵活，但是Map不能直接转换为JSON格式进行通讯。可以把Map作为Object的补充来使用。

    冒泡，捕获，委托
    冒泡

    当父子节点均设置了监听事件后，其有从子到父的顺序关系，为事件冒泡
    以点击事件为例：

    1. 点击触发子元素的事件
    2. 冒泡找到父节点，执行父节点的监听事件
    3. 以此冒泡直到html根元素

    设置e.stopPropagation()可以阻止冒泡

    捕获

    捕获与冒泡相反，其从根节点到子节点依次移动

    委托

    委托是冒泡的一种应用，其的实现基于一个事实：在大量子元素中点击任何一个都会执行一段代码，实现这个可以将监听事件放在父元素上，通过冒泡来找到相应父元素的执行代码。

    鼠标移动事件：

    1. mousemove：
      鼠标移动事件，在元素内移动时持续触发
    2. mouseenter/mouseleave：
      鼠标进入和离开事件，但只作用于当前DOM节点，不会作用于其后代节点

      鼠标进入和离开事件，但是会作用于其后代节点
