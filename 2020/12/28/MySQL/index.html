CRUD

表格的结构

关系数据库的表格由以下元素组成：

  • 表由表名，行，列，列名构成
  • 表名是表的名称
  • 列名表示列的名字，列名不可以重复
  • 表格实质上是一个二维数组，行列都是从0开始

表名

一般用英文小写字母来表示表名，多个单词使用_分割。在数据库中，将表名和字段名都小写，数据库操作语句都大写。

字段

在数据库表中，每一列都是一个字段，第一行是字段名，下面都是字段的值，从第二行开始，对应序号为0。

字段名不能重复，字段名用来约定字段的值，或者和其它的表产生联系。字段的值可以为NULL，但应尽量避免。

主键

每一张数据库表都应有一个主键(primary key)，主键用来标识数据。其有以下特点：

  • 主键是一个特殊字段
  • 表格可以没有主键，但是最多只能有一个主键
  • 主键的值不能为NULL，必须有值
  • 主键的值必须是绝对唯一的
  • 一般使用主键和其它的表进行关联
  • 主键也可以由多个字段组合而成

SQL常用数据类型

类型 含义
VARCHAR 可变的长字符串，类比JAVA中的String
INT 整形，类比JAVA中的int
DOUBLE 浮点型，类比JAVA中的double，一般不加长度限制
DATETIME 时间类型，长度为0，格式为YYYY-MM-DD HH:MM:SS
BIGINT 长整型，类比JAVA中的long

数据

将表格中的一行称为数据，可以通过主键或其它字段操作数据。

NULL值的处理

当查询语句的条件字段为NULL时，该命令可能无法正常工作。

NULL值的概念

NULL以及NOT NULL，他们是MySQL字段的数据类型的属性，本身不属于任何数据类型，NULL值也不等于空值，空值是不占用存储空间的，但是NULL占用。

如何处理NULL值

MySQL为处理NULL值提供了三种运算符：

  1. IS NULL：当列的值是NULL时，返回true
  2. IS NOT NULL：当列的值不为NULL时，返回true
  3. <=>：比较操作符，当左右两边的值均为NULL或者相等时，返回true

不能使用=NULL或者!=NULL来查找NULL，NULL = NULL会返回NULL。

TRIM函数

用户输入数据的时候可能会含有空格等不需要的字符，产生脏数据，为了保证数据格式的正确性，可以使用TRIM函数来清理数据

语法

TRIM函数只是改变查询结果，不改变原数据，改变原数据需要UPDATE/DELETE

语法拓展

TRIM函数也可以精准去除前面或者和后面的空格或者其它字符

  • 加上LEADING只去除前面的空格，加上TRAILING只去除后面的空格，默认BOTH
  • TRIM可以删除指定的字符串，如果不加默认删除空格

REPLACE函数

TRIM只能删除两边的字符，不能删除中间的，可以使用REPLACE函数做到删除中间的字符，但是实用性不高，可以直接使用UPDATE修改

MySQL关联查询

左连接

语法

  • JOIN是关联查询的关键词，基础结构是TableA JOIN TableB，即A表和B表关联查询，LEFT表示左连接
  • ON后面是关联条件
  • 左连接是指返回左表的所有数据，即使右表中没有匹配的数据(此时以NULL的形式匹配数据)

表结构的设计规范

  • 表必须要有主键
  • 一个字段只表示一种含义
  • 总是包含创建时间和修改时间这两个字段，且不包含其它的业务逻辑，类型使用DATETIME
  • 禁止使用复杂数据类型，如数组，自定义类型
  • 禁止使用物理外键，使用逻辑外键
  • 禁止物理删除，使用逻辑删除

最后两点，对于逻辑外键，比如表b记录了表a的id，只需要在表b中添加一列a_id即可，然后通过程序来控制外键关系。对于逻辑删除，是指为数据添加某些标识表示其被删除，而不是直接将其从数据库中删除。
