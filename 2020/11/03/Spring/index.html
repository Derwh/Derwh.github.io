Spring核心

Spring是java语言必须要掌握的框架，其核心技术点是依赖注入(DI)。使用Spring能大大降低开发难度和助力团队发展，更加强调面向对象。

Maven

Maven简介

Maven，全称Apache Maven，其是一个项目管理和构建自动化工具，其最被关注的功能是项目构建功能，Maven是必须掌握的工具。Maven提供了一个命令行工具可以把工程打包成java支持的格式(比如jar)，并且支持部署到中央仓库，如此，使用者可以很方便运用别人的代码，只需要添加依赖即可。
Maven系统构架图：

Maven使用惯例优于配置的原则，在没有定制之前，所有的项目要有如下的结构：

目录 目的
${basedir} 存放pom.xml和所有的子目录
${basedir}/pom.xml Maven项目配置文件
${basedir}/src/main/java 项目的java源代码
${basedir}/src/main/resources 项目的资源，比如property文件
${basedir}/src/test/java 项目的测试类，比如说JUnit代码
${basedir}/src/test/resources 测试使用的资源

这里的${basedir}代表的是工程的根路径。一个Maven项目在默认情况下会产生JAR文件，编译后的classes会放在${basedir}/target/classes下面，JAR文件会放在${basedir}/target下面。代码位置很重要，放错会导致不能运行！

Maven安装

Maven本地安装参考：Maven安装

Maven命令

这个Maven工具是在命令行(终端软件)里输入指令的方式在执行的，需要掌握以下常见的命令，命令要在工程的根目录下执行。

  1. mvn clean compile
    编译命令，Maven自动扫描src/main/java下面的代码并完成编译工作，执行完会在根目录下生成target/classes目录。

  2. mvn clean package
    编译并打包命令，其是compile和package的集合，就是说会先执行compile命令，结果会把所有的java文件和资源打包成一个jar格式的压缩包。

  3. mvn clean install
    执行安装命令，其是complie，package和install的集合，就是说会先执行compile命令，再执行打包命令，最后执行安装到本地Maven仓库目录里的命令。这个目录是${basedir}.m2。${user_home}是电脑登录用户名的个人目录。

  4. mvn compile exec:java -Dexec.mainClass=${main}
    命令意思是执行了compile后，执行运行java的命令，具体执行哪个是${main}这个参数决定的。比如：mvn compile exec:java -Dexec.mainClass=com.youkeda.Test

上述命令在idea上可能无法执行，在ieda上执行命令，可以直接点击右边Maven按钮，上面m图标即可。

Maven坐标

这四个标签决定了Maven的坐标，所谓坐标就是Maven的位置信息，其决定了Maven工程部署后在Maven仓库中的文件位置，所以其必须要指定。

  • groupId
    其就像一个文件夹，命名和java包相似，以小写字母和.组成

  • artifactId
    其像一个文件名，在groupId中应该是唯一的。只能使用小写英文字母、.、-、_

  • packaging
    Maven工程执行完后，会把整个工程打包成packaging指定的格式，默认情况下是jar，若pom.xml中没有声明这个属性，就是jar，除了jar以外，还有别的格式：

    • war
    • ear
    • pom

  • version
    版本号，其基本上遵守软件工程中对版本号的要求

    在Maven中，会把工程分为两个状态：

    • SNAPSHOT
      译为快照，表示当前工程是出于不稳定阶段，在开发时在版本号后加上。
    • RELEASE
      其代表是稳定，在正式发布时改为RELEASE，当然不用特意加上。

    三位版本号的使用也是有规则的：

    • 第一位代表主版号，一般是开发团队约定的
    • 第二位时新增功能，每次新增功能后增加
    • 第三位时bugfix后的版本，bugfix是修复代码功能的行为，数字代表修复了多少次bug

    也有时只有两位版本号，没有第一位的主版号。软件的版本号从1.0.0开始。一般来说每一位不超过100。

    最后，执行Maven命令生成的jar文件名格式是[artifactId]-[version].jar。

Maven依赖

有了上述的Maven坐标，我们就可以通过Maven依赖管理来运用别人的代码了。依赖管理的标签是dependencies，其就是用于指定当前工程依赖其它代码库的，Maven会自动管理jar依赖。当在pom.xml文件中声明dependencies标签后，会先去本地用户目录下的.m2文件夹中去寻找对应文件，如果没有就从中央仓库下载。下载完会保存在.m2文件夹中。

一个pom.xml只能有一个dependencies，但其中间可以有多个dependency，每个代表一个依赖。标签其中实际上是Maven坐标信息，也就是说有了坐标信息就可以建立依赖了。

一般别人的代码库被称为三方库，自己或团队写的为二方库

中央仓库

Maven会将所有的jar存在中央仓库中，可以通过这个链接访问中央仓库，不能翻墙的情况下，可以使用阿里云的镜像服务器。

间接依赖

间接依赖是mvn成功的核心要素，简单的说，如果一个remote工程依赖了okhttp库，而当前工程locale依赖了remote工程，那么它也依赖了okhttp库。

Spring依赖注入

java注解(Annotation)

上例中@Service就是一个注解。本质上，注解是java推出的一种注释机制，然而和普通的注释相比较，它可以在编译，运行阶段读取的。其给了用户很多拓展空间，并且不会污染源代码，可以利用它实现一些增强功能。Spring中大量使用了注解。

可以通过@Service的源码来理解注解：

定义注解是要使用别的注解的，那些被使用在注解里面的注解被称为元注解，下面对图中各处标号解释：

  1. @Target
    其有一个数组属性，用于注明注解的使用范围，可以同时设定多个。它的值可以是：

    • ElementType.TYPE
      表示可以使用在类，接口类，枚举类上

    • ElementType.FIELD
      表示可以使用在类的属性上

    • ElementType.METHOD
      表示可以使用在类的方法上

    • ElementType.PARAMETER
      表示可以使用在类的参数上

      若要设置多个作用范围可以这样：

      @Target({ElementType.TYPE,ElementType.METHOD})

  2. @Retention
    用于声明注解的生命周期，就是在java编译运行的哪个环节有用。它的值定义在java.lang.annotation.RetentionPolicy枚举类中，有可选的三个值：

    • SOURCE：纯注释作用
    • CLASS：在编译阶段有效

    • RUNTIME：在运行时有效

      例如：@Retention(RetentionPolicy.RUNTIME)，如果时自定义的话，一般设置为RUNTIME。

  3. @Documented
    作用是将注解中的元素包含到JavaDoc文档中，一般都会添加这个注解。

  4. @interface
    声明当前类为注解，固定写法。

  5. Anotation属性
    String value() default "";
    和类的属性相似，定义了属性的类型(基础类型)，属性名称(默认value，使用时可以忽略)，默认值(default)。

Spring Bean

IoC(控制反转)容器是Spring框架的最核心组件，其是面向对象编程中的一种设计原则，是一种思想为非技术。
在Spring框架中主要通过依赖注入(DI)来实现IoC。
在Spring中，所有的Java对象都会通过IoC容器转变为Bean(Spring对象的一种称呼，用Bean来表示对象)。构成应用程序主干和由Spring IoC容器管理的对象称为beans，beans和它们的之间的依赖关系反映在容器使用的配置原数据中。基本上所有的Bean都是由接口+实现类完成，想获取实例直接从IoC容器中获取即可。
Spring有两种配置元数据的方式，一种是基于XML，一种是基于Annotation，主流使用第二种。

org.springframework.context.ApplicationContext接口类定义容器的对外服务，通过这个接口可以获得IoC容器中的Bean对象。Annotation类型的IoC容器对应的类是org.springframework.context.annotation.AnnotationConfigApplicationContext。

Spring Resource

在Spring中，对文件的处理需要依靠Spring Resource

java工程中文件的几种情况：

  1. 文件在电脑某个位置，比如：d:/mywork/a.doc
  2. 文件在工程目录下，比如：mywork/toutiao.png
  3. 文件在工程的src/main/resources目录下，就是Maven工程存放文件的地方

第一种情况和第二种情况可以直接使用File进行文件的读写，但是第三种情况比较特殊，因为Maven执行package时，会把resources目录下的文件一起打包到jar包里，所以File读取不到。

classpath

在java内部中，将文件路径称为classpath，因此读取内部文件就是从classpath内读取，classpath指定的文件不能解析为File对象，但是可以解析为InputStream，可以借助javaIO读取。

Spring MVC

Spring Boot介绍

Spring Boot是一个新的面向微服务的框架，其更像是一个方案，Spring工程的快速上手方案，极大地降低了Java Web工程的创建，运行和部署的难度。Spring Boot的核心还是Spring。Spring Boot集成了Tomcat，所以不用额外部署这类服务器，其自定义了工程打包格式，通过这个可以把Java Web工程转为普通的Java工程，降低了开发难度。其默认集成了很多第三方框架和服务，强调了开箱即用这个概念。

可以通过访问这个网站来创建Spring Boot工程。

  • it.current
    当前迭代变量，等同于上面的song

  • it.even/odd
    布尔值，当前循环是奇数还是偶数(从0开始计算

  • it.first
    布尔值，当前循环是否是第一个

  • it.last
    布尔值，当前循环是否是最后一个
